Intelligent Error Analysis

Leverage AI and vector search to automatically categorize and analyze test failures with unprecedented accuracy.

How It Works

Our AI pipeline uses Ollama with GPT models and vector embeddings to analyze error patterns, search through 60,000+ Jira issues, and classify failures into actionable categories.

1

Jira Knowledge Base

We fetch and process 60,000+ Jira issues (MB-926 to MB-69610), clean the descriptions, map components, and generate vector embeddings using the nomic-embed-text model.

fetch_jira_issues.py ‚Üí Couchbase "issues" bucket with vector embeddings
2

Error Extraction & Categorization

For each build, we extract claims from failed jobs and use AI to categorize them into descriptive error types like "Initial rebalance failed" or "AssertionError: Value mismatch".

analyze_errors.py ‚Üí Two-pass categorization with merging
3

Vector Search for Context

Each error type is embedded and compared against the Jira knowledge base using COSINE similarity to find the most relevant historical issues for context.

APPROX_VECTOR_DISTANCE(embedding, query_vector, 'COSINE') < 1.0
4

Classification with Labels

Using the Jira context, each error is classified into one of three categories:

Product Issue Testware Issue Environmental Issue
